# AdminDB Skill — Full-Stack Implementation Guide

This skill encodes the exact patterns, conventions, and quality bar for the AdminDB project.
Read the relevant reference file before writing code. Never guess at conventions — they are specified here.

## Project Overview

Admin dashboard where authenticated admins can:
1. Sign up / sign in (email + password via Supabase Auth)
2. Create organizations of 3 types (School, Nonprofit, Business) with type-conditional fields
3. Invite members by email to any org they own (via Edge Function)
4. View all their orgs and click into each to see the member list

**Tech stack (non-negotiable):**
- Frontend: React 18 + TypeScript (strict) + Vite (SWC) + React Router v6
- Styling: Tailwind CSS + shadcn/ui (Radix-based). No second component library.
- State: TanStack React Query for all server state. No raw `useEffect` + fetch for data loading.
- Forms: React Hook Form + Zod. Every form uses both.
- Icons: Lucide React only.
- Backend: Supabase (Postgres + Auth + Edge Functions in Deno)
- Dark mode: next-themes

## Non-Negotiable Quality Gates

Every piece of code produced for this project must pass these checks before being considered done:

1. **TypeScript strict** — zero `any`, zero `@ts-ignore`, every prop typed
2. **No raw fetch in components** — all server state via React Query hooks
3. **No useEffect for data loading** — that's React Query's job
4. **Every form uses RHF + Zod** — no uncontrolled inputs, no manual validation
5. **Loading + empty + error states** — every data-dependent UI must handle all three
6. **RLS on every table** — no table without a policy; service-role key never in client bundle
7. **Edge Function auth** — every Edge Function verifies JWT and returns 401 if invalid
8. **shadcn/ui components** — use existing shadcn primitives; don't hand-roll what's available
9. **Lucide icons only** — no other icon library
10. **Dark mode safe** — every color via Tailwind semantic class or CSS variable; no hardcoded hex

## Org Types and Conditional Fields

| Type | Extra required field | Label | Validation |
|---|---|---|---|
| `school` | `schoolDistrict` | School District | min 1 char |
| `nonprofit` | `nonprofitEin` | EIN Number | pattern `/^\d{2}-\d{7}$/` |
| `business` | `businessRegNumber` | Registration Number | min 1 char |

Type badge colors (Tailwind):
```ts
const TYPE_BADGE: Record<OrgType, { bg: string; text: string }> = {
  school:    { bg: 'bg-blue-100 dark:bg-blue-900/30',    text: 'text-blue-800 dark:text-blue-300' },
  nonprofit: { bg: 'bg-teal-100 dark:bg-teal-900/30',    text: 'text-teal-800 dark:text-teal-300' },
  business:  { bg: 'bg-amber-100 dark:bg-amber-900/30',  text: 'text-amber-800 dark:text-amber-300' },
}
```

## Environment Variables

Client-side (prefixed `VITE_`):
```
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
```

Edge Function secrets (Supabase secrets, never in client bundle):
```
SUPABASE_SERVICE_ROLE_KEY=   # only used server-side
```

Never commit `.env`. Always keep `.env.example` updated.
